# cin & cout

cin read until a whitespace but not consume it. it’s still in the buffer.

when the buffer has no left, the program prompt the user for input.(cin)

when cin want to read an int interger, it failed and fail bit is on, so cannot read others

why cin with >> is a nightmare?

  • cin reads the entire line into the buffer but extracts whitespace-separated tokens.
  • Trash in the buffer will make cin not prompt the user for input at the right time
  • When cin fails, all future cin operations fall too.

getline(cin, name) instead of cin >> name can read the entire line. getline reads up to the next newline character.

when to use auto?

  • when you don’t care what the type is( iterators)
  • when it’s type is clear from context (templates)
  • when yoiu don’t know what the type is (lambda)
  • don’t use it unnecessarily for return types
