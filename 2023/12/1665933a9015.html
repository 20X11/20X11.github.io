# Const keywords

The keyword const : just like a const reference parameter can’t be modified by the function.

void foo (const BankAccount &ba)

A const member function can’t change the object’s state

const member function & unconst member function

void foo (const Object& input)

void bar(Object& input)

The const Model

A const Pointer

**const iterators **

Remember that iterators act like pointers

const vector<int>::iterator itr however, acts like int * const itr

To make an iterator read only, define a new const_iterator

summary

  • const on objects : Guarantee that the object won’t change by allowing you to call you only const functions and treating all public members as if they were const. This helps the programmer write safe code, and also give the compiler more information to use to optimize.
  • const on functions : Guarantees that the function won’t call anything but const functions, and won’t modify any non-static, non-mutable members.

# special member functions

  • Default construction: object created with no parameters

  • Copy construction: object is created as a copy of an existing object

  • Copy assignment : existing object replaced as a copy of another existing object

  • Destruction :object destroyed when it’s out of scope
