# Sequence Containers

Provides access to sequences of elements.

# std::vector<T>

A vector represents a sequence of elements of any type

You specify the type when using the vector

vec.at(i) throws an exception

vec[i] causes undefined behaviour

Deque or vector?

vector is the type of sequence that should be used by default, deque is the data structure of choice when most insertions and deletions take place at the beginning or at the end of the sequence.

# Container Adaptors

How to implement stack and queue using the containers we have?

stack : Just limit the functionality of a vector/deque to only allow push_back and pop_back

queue : Just limit the functionality of a deque to only allow push_back and pop_front

for this reason, stack and queue are known as container adaptors. They are just adapt vector and deque.

Why not just use a vector / deque?

Design philosophy of c++:

  • Allow the programmer full control, responsbility, and choice if they want it.
  • Express ideas and intent directly in code.
  • Enforce safety at compile time wheneverpossible.
  • Do not waste time or space
  • Compartmentalize messy constructs.

# Associative Containers

Data is accessed using the key instead of indexes.

map & set based on ordering property of keys

keys need to be comparable using < (less than) operator

map & set : Keys in sorted order, faster to iterate through a range of elements

unordered_map & unordered_set : faster to access individual elements by key.

multimap

# Iterators

Iterators allow iteration over any container, whether it is ordered or not

Iterators let us view a non-linear collection in a linear manner.

set<int>::iterator iter = mySet.begin();

cout << *iter << endl;

if (iter == mySet.end()) return;
