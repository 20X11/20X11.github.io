# 位操作

  • | 用于置 1 P0 = P0 | 0x0F //低4位置1

  • & 用于置 0 P0 = P0 & 0xFE //最低位清0

  • ~ 用于取反

# 存储区

# data

固定指前面 0x00~0x7F 的 128 个 RAM，可以用 ACC 直接读写，速度最快，生成的代码也最小。 变量默认为 data 型

# bdata

指被分配在 20H~2FH 的 16B 的存储位置，可以位寻址

# idata

固定指前面 0x00~0xFF 的 256 个 RAM， 其中前 128 个和 data 的 128 完全相同，只是访问的方式不同。

# pdata

将被分配在片外数据存储器低 266B 中，编译成 MOVX A, R0 这类方式进行访问

# xdata

外部扩展 RAM， 一般指外部 0x0000~0xFFFF 空间，用 DPTR 访问

# code

ROM，写入后不能再更改
code 型数据表示存储于程序存储器中的常数，可以是数据表格，字库等掉电后还需要保存的数据，通常以数组的形式定义

# 变量定义方法

[存储种类] + 数据类型 + [存储器类型] + 变量名表 + [_at_常量表达式]

uchar xdata tmp[10]

uchar xdata * data pstr : 在 RAM 区分配一个指针变量，指针本身指向 xdata 区

# 常量定义方法

  • 宏定义： #define False 0

  • 用 const 和 code 来定义 ：

    unsigned int code a = 100 用存储类型 code 把 a 定义在程序存储器中并赋值
    const unsigned int c = 100 用 const 定义 c 为无符号 int 常量并赋值

    unsigned char code x[] = {0x00, 0x01, 0x02};

# 关键字 volatile

表示可以被某些编译器未知的因素更改

对于单片机， volatile 变量可能用于如下几种情况：

  • 并行设备的硬件寄存器（如：状态寄存器）
  • 一个中断服务支持下中会访问到的非自动变量（即全局变量）
  • 多线程应用中被几个任务共享的变量

# 中断服务程序 ISR

当中断发生时， C51 编译器提供一个调用 c 函数的方法，这可以使用户可以使用 c 创建中断服务程序

在 keil C 中常用的中断函数说明

在进入中断函数时，特殊功能寄存器 ACC， B， DPTR， PSW 将被保存入栈

如果不使用寄存组切换，则将中断函数中所用到的全部工作寄存器都入栈

void function() interrupt n [using m];

using m 可以指定中断函数所使用的寄存器组。中断响应时，默认的工作寄存器组就不会进入堆栈，节省时间。 缺点：所有被中断调用的函数都必须使用同一个寄存器组，否则参数传递会发生错误

函数返回前，所有的寄存器内容出栈。

ISR 不能进行参数传递，没有返回值。在任何情况下都不能直接调用 ISR

如果中断函数中用到了浮点运算，必须保存浮点寄存器的状态，可以使用 math.h 中的保存浮点寄存器的 fpsave 和恢复浮点寄存器的 fprestore

如果在中断函数中调用了其他函数，则被调用函数所使用的寄存器组必须与中断函数相同

# TFT-LCD 显示屏

TFT-LCD 是有源矩阵类型液晶，每个像素是在像素电极和共同电极间夹一层液晶（相当于一个电容）LCD 背部的发光管可以” 主动 “对屏幕上各个独立像素进行控制

TFT 的色彩饱和度，还原能力和对比度较好，但耗电大，成本较高

不能由单片机直接驱动，需要专用芯片控制

本实验所用 TFT 分辨率 320x240， 支持 RGB365 彩色显示

TFT-LCD 通过 ILI9325 驱动芯片与单片机连接。CPU 通过读写 ILI9325 的寄存器和片内显示 RAM 来实现对液晶的访问控制
