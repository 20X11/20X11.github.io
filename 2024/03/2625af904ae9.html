# 1. 总线的基本概念

总线是连接多个部件的信息传输线，是各部件共享的传输介质。
在某一时刻，只允许一个部件向总线发送信息，而多个部件可以同时从总线上接受相同的信息

  • 单总线结构框图

    I/O 设备与主存交换信息时，原则上不影响 CPU 的工作，CPU 仍可处理不访问主存和 I/O 的操作，提高了 CPU 的工作效率。

    只有一组总线，容易发生冲突，需要设置总线判优逻辑，会影响工作速度

  • 双总线结构框图

    CPU 和主存之间为存储总线（M 总线），CPU 与 IO 设备之间为输入输出总线（I/O 总线），各种 IO 设备通过 IO 接口挂到 IO 总线上，便于增删设备。但是在 I/O 设备与主存交换信息时仍然要占用 CPU，会影响 CPU 的工作效率


以存储器为中心的双总线结构，在单总线基础上加上了一条 CPU 和主存之间的存储总线，既提高了传输效率，又减轻了系统总线的负担，还保留了 IO 设备与存储器交换信息时不经过 CPU 的特点。

# 2. 总线的分类

# 片内总线：

芯片内部的总线，在 CPU 芯片内部，寄存器之间，寄存器与算术逻辑单元之间都是片内总线

# 系统总线：

系统总线指 CPU，主存，IO 设备各大部件之间的总线。又称板级总线 / 板间总线

  • 数据总线：
    双向，位数与机器字长，存储字长有关，一般为 8，16，32 位
    数据总线的位数称为 数据总线宽度，是衡量系统性能的重要参数。
    如果数据总线的宽度是 8 位，指令字长是 16 位，CPU 在取指阶段必须两次访问内存。

  • 地址总线：
    用来指出数据总线上的源数据或目的数据在主存单元或 IO 设备上的地址，由 CPU 输出，单向传输。

    存储单元的个数=2地址线的位数存储单元的个数=2^{地址线的位数}存储单元的个数=2地址线的位数
    .

  • 控制总线：
    用来发出各种信号，对于控制线来说，传输是单向的，对于控制总线总体来说，又是双向的。对于 CPU 来说，控制信号既有输入，又有输出。
